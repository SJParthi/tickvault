# Dhan V2 Live Market Feed WebSocket — Complete Reference

> **Source**: https://dhanhq.co/docs/v2/live-market-feed/
> **Extracted**: 2026-03-13
> **Purpose**: Standalone reference for Claude Code — no URL fetching needed
> **Related files**: `08-annexure-enums.md` (enums, error codes, rate limits), `09-instrument-master.md` (SecurityId lookup)

---

## 1. Overview

- Real-time **tick-by-tick event-based** market data via WebSocket
- All responses are **binary** (Little Endian byte order)
- All requests are **JSON**
- Requires a WebSocket library + binary parser in your language

### Connection Limits

- **Max 5 WebSocket connections** per user
- **Max 5000 instruments** per connection
- **Max 100 instruments** per single JSON subscribe message (send multiple messages to reach 5000)

---

## 2. Establishing Connection

### WebSocket Endpoint

```
wss://api-feed.dhan.co?version=2&token=<ACCESS_TOKEN>&clientId=<CLIENT_ID>&authType=2
```

### Query Parameters

| Parameter   | Required | Value                                    |
|-------------|----------|------------------------------------------|
| `version`   | Yes      | `2` (DhanHQ v2)                          |
| `token`     | Yes      | Access Token from Dhan                   |
| `clientId`  | Yes      | User-specific ID generated by Dhan       |
| `authType`  | Yes      | `2` (always)                             |

## 3. Subscribing to Instruments

### Request JSON Structure

```json
{
    "RequestCode": 15,
    "InstrumentCount": 2,
    "InstrumentList": [
        {
            "ExchangeSegment": "NSE_EQ",
            "SecurityId": "1333"
        },
        {
            "ExchangeSegment": "BSE_EQ",
            "SecurityId": "532540"
        }
    ]
}
```

### Request Fields

| Field                              | Type        | Description                                                                 |
|------------------------------------|-------------|-----------------------------------------------------------------------------|
| `RequestCode`                      | int         | Feed request code — determines data mode (see 08-annexure Section 2)        |
| `InstrumentCount`                  | int         | Number of instruments in this message (max 100)                             |
| `InstrumentList.ExchangeSegment`   | enum string | Exchange segment string enum (see 08-annexure Section 1)                    |
| `InstrumentList.SecurityId`        | string      | Exchange standard ID from instrument master CSV (see 09-instrument-master)  |

### Common Request Codes for this WebSocket

| Code | Mode             |
|------|------------------|
| `15` | Subscribe Ticker |
| `16` | Unsubscribe Ticker|
| `17` | Subscribe Quote  |
| `18` | Unsubscribe Quote|
| `21` | Subscribe Full   |
| `22` | Unsubscribe Full |
| `12` | Disconnect       |

> **CRITICAL**: `SecurityId` values come from the instrument master CSV — it is the sole source of truth. Do NOT hardcode or guess these values.

---

## 4. Keeping Connection Alive

- Server sends **ping every 10 seconds**
- WebSocket library auto-responds with pong
- If client does not respond for **>40 seconds**, server closes the connection
- You must handle reconnection logic in your code

---

## 5. Binary Response Format

All market data responses are binary, Little Endian.

Every response = **Response Header (8 bytes)** + **Payload (variable)**

### Endianness

- **Little Endian** — least significant byte first
- If your system is Big Endian, you must convert while reading

---

## 6. Response Header (8 bytes — common to ALL responses)

| Byte Position | Type     | Size (bytes) | Description                                                     |
|---------------|----------|--------------|-----------------------------------------------------------------|
| `1`           | byte     | 1            | Feed Response Code (see 08-annexure Section 3)                  |
| `2-3`         | int16    | 2            | Message Length of entire payload packet                          |
| `4`           | byte     | 1            | Exchange Segment numeric code (see 08-annexure Section 1)       |
| `5-8`         | int32    | 4            | Security ID                                                     |

**Total header: 8 bytes**

---

## 7. Data Packets

### 7.1 Ticker Packet (Response Code `2`)

Smallest packet. LTP + LTT only.

| Byte Position | Type    | Size (bytes) | Description                        |
|---------------|---------|--------------|------------------------------------|
| `0-8`         | array   | 8            | Response Header (code `2`)         |
| `9-12`        | float32 | 4            | Last Traded Price (LTP)            |
| `13-16`       | int32   | 4            | Last Trade Time (LTT) — UNIX EPOCH |

**Total: 16 bytes**

---

### 7.2 Prev Close Packet (Response Code `6`)

Sent automatically whenever ANY instrument is subscribed (any mode). Provides previous day's close data.

| Byte Position | Type    | Size (bytes) | Description                          |
|---------------|---------|--------------|--------------------------------------|
| `0-8`         | array   | 8            | Response Header (code `6`)           |
| `9-12`        | float32 | 4            | Previous day closing price           |
| `13-16`       | int32   | 4            | Open Interest — previous day         |

**Total: 16 bytes**

> **CRITICAL**: This packet arrives on *every* subscription, not just quote/full mode. Your parser must handle response code `6` regardless of which mode you subscribed with.

---

### 7.3 Quote Packet (Response Code `4`)

Complete trade data without market depth.

| Byte Position | Type    | Size (bytes) | Description                                        |
|---------------|---------|--------------|----------------------------------------------------|
| `0-8`         | array   | 8            | Response Header (code `4`)                         |
| `9-12`        | float32 | 4            | Last Traded Price (LTP)                            |
| `13-14`       | int16   | 2            | Last Traded Quantity                                |
| `15-18`       | int32   | 4            | Last Trade Time (LTT) — UNIX EPOCH                 |
| `19-22`       | float32 | 4            | Average Trade Price (ATP)                           |
| `23-26`       | int32   | 4            | Volume                                              |
| `27-30`       | int32   | 4            | Total Sell Quantity                                 |
| `31-34`       | int32   | 4            | Total Buy Quantity                                  |
| `35-38`       | float32 | 4            | Day Open                                            |
| `39-42`       | float32 | 4            | Day Close (only sent **post market close**)         |
| `43-46`       | float32 | 4            | Day High                                            |
| `47-50`       | float32 | 4            | Day Low                                             |

**Total: 50 bytes**

---

### 7.4 OI Data Packet (Response Code `5`)

Sent automatically when subscribed to Quote mode. Important for derivative contracts.

| Byte Position | Type  | Size (bytes) | Description                    |
|---------------|-------|--------------|--------------------------------|
| `0-8`         | array | 8            | Response Header (code `5`)     |
| `9-12`        | int32 | 4            | Open Interest of the contract  |

**Total: 12 bytes**

> **NOTE**: This is a separate packet from the Quote packet. When subscribed to Quote mode, you receive BOTH code `4` (quote) and code `5` (OI) as separate binary messages.

---

### 7.5 Full Packet (Response Code `8`)

Everything: trade data + OI + 5-level market depth in a single packet.

| Byte Position | Type                  | Size (bytes) | Description                                         |
|---------------|-----------------------|--------------|-----------------------------------------------------|
| `0-8`         | array                 | 8            | Response Header (code `8`)                          |
| `9-12`        | float32               | 4            | Last Traded Price (LTP)                             |
| `13-14`       | int16                 | 2            | Last Traded Quantity                                 |
| `15-18`       | int32                 | 4            | Last Trade Time (LTT) — UNIX EPOCH                  |
| `19-22`       | float32               | 4            | Average Trade Price (ATP)                            |
| `23-26`       | int32                 | 4            | Volume                                               |
| `27-30`       | int32                 | 4            | Total Sell Quantity                                  |
| `31-34`       | int32                 | 4            | Total Buy Quantity                                   |
| `35-38`       | int32                 | 4            | Open Interest (Derivatives only)                     |
| `39-42`       | int32                 | 4            | Highest OI for the day (NSE_FNO only)                |
| `43-46`       | int32                 | 4            | Lowest OI for the day (NSE_FNO only)                 |
| `47-50`       | float32               | 4            | Day Open                                             |
| `51-54`       | float32               | 4            | Day Close (only sent **post market close**)          |
| `55-58`       | float32               | 4            | Day High                                             |
| `59-62`       | float32               | 4            | Day Low                                              |
| `63-162`      | Market Depth (5x20B)  | 100          | 5 levels of market depth (see below)                 |

**Total: 162 bytes**

#### Market Depth Structure (20 bytes per level × 5 levels)

| Byte Offset (within level) | Type    | Size (bytes) | Description       |
|-----------------------------|---------|--------------|-------------------|
| `1-4`                       | int32   | 4            | Bid Quantity      |
| `5-8`                       | int32   | 4            | Ask Quantity      |
| `9-10`                      | int16   | 2            | No. of Bid Orders |
| `11-12`                     | int16   | 2            | No. of Ask Orders |
| `13-16`                     | float32 | 4            | Bid Price         |
| `17-20`                     | float32 | 4            | Ask Price         |

> **Implementation Note**: Bytes 62-161 (0-based) contain 5 consecutive 20-byte depth structures. Parse as: `for level in 0..5 { offset = 62 + (level * 20); ... }`

## 8. Feed Disconnect

### Client-Initiated Disconnect

```json
{
    "RequestCode": 12
}
```

### Server-Initiated Disconnect (Response Code `50`)

| Byte Position | Type  | Size (bytes) | Description                                       |
|---------------|-------|--------------|---------------------------------------------------|
| `0-8`         | array | 8            | Response Header (code `50`)                       |
| `9-10`        | int16 | 2            | Disconnection reason code (see 08-annexure Section 11)|

**Key disconnect code**: `805` — sent when >5 WebSocket connections exist; first (oldest) socket gets disconnected.

## 10. Critical Implementation Notes

1. **LTT timestamps are standard UNIX epoch (UTC-based).** You MUST add +5:30 (19800s) to get IST. See 08-annexure Section 12 for full verification.
   ```rust
   use chrono::{FixedOffset, TimeZone};
   let ist = FixedOffset::east_opt(5 * 3600 + 30 * 60).unwrap();
   let dt = ist.timestamp_opt(ltt as i64, 0).unwrap();
   ```

2. **Byte ordering is Little Endian** — all multi-byte fields (int16, int32, float32) must be read as LE. In Rust: `f32::from_le_bytes()`, `i32::from_le_bytes()`, etc.

3. **Byte indexing in docs is 1-based** — Dhan docs use 1-based byte positions. In code (0-based arrays), subtract 1. E.g., "bytes 9-12" in docs = `buffer[8..12]` in code.

4. **Response code `6` (Prev Close) arrives on ANY subscription** — even Ticker mode. Your binary dispatcher must handle all response codes.

5. **OI packet (code `5`) is separate from Quote packet (code `4`)** — subscribing to Quote gives you TWO different packet types.

6. **Day Close (in Quote and Full packets) is only valid post-market** — during market hours this field may be zero or stale.

7. **OI fields in Full Packet**: `highest_oi` and `lowest_oi` (bytes 39-46) are only populated for NSE_FNO. Other segments = zero.

8. **Max 100 instruments per JSON message** — to subscribe to 5000, send 50 separate messages.

9. **Ping-pong timeout is 40 seconds** — keep WebSocket handling on a dedicated async task.

10. **Disconnect code `805`** — 6th WebSocket kills the 1st (oldest). Track active connections.

11. **Exchange Segment in binary header (byte 4) is NUMERIC**, not string. See 08-annexure Section 1 for the mapping.

## 11. Parsing Pseudocode (Rust-style)

```rust
fn parse_market_feed(buffer: &[u8]) -> Result<MarketFeedMessage, ParseError> {
    if buffer.len() < 8 {
        return Err(ParseError::TooShort);
    }

    let response_code = buffer[0];
    let msg_len = i16::from_le_bytes([buffer[1], buffer[2]]);
    let exchange_seg = buffer[3];
    let security_id = i32::from_le_bytes([buffer[4], buffer[5], buffer[6], buffer[7]]);

    match response_code {
        2 => {
            // Ticker: 16 bytes total
            let ltp = f32::from_le_bytes(buffer[8..12].try_into()?);
            let ltt = i32::from_le_bytes(buffer[12..16].try_into()?);
            Ok(MarketFeedMessage::Ticker { security_id, ltp, ltt })
        }
        4 => {
            // Quote: 50 bytes total
            let ltp = f32::from_le_bytes(buffer[8..12].try_into()?);
            let ltq = i16::from_le_bytes(buffer[12..14].try_into()?);
            let ltt = i32::from_le_bytes(buffer[14..18].try_into()?);
            // ... parse remaining fields
            Ok(MarketFeedMessage::Quote { /* fields */ })
        }
        5 => {
            // OI: 12 bytes total
            let oi = i32::from_le_bytes(buffer[8..12].try_into()?);
            Ok(MarketFeedMessage::OI { security_id, oi })
        }
        6 => {
            // Prev Close: 16 bytes total
            let prev_close = f32::from_le_bytes(buffer[8..12].try_into()?);
            let prev_oi = i32::from_le_bytes(buffer[12..16].try_into()?);
            Ok(MarketFeedMessage::PrevClose { security_id, prev_close, prev_oi })
        }
        8 => {
            // Full: 162 bytes total
            // ... parse all fields including 5 depth levels
            // for level in 0..5 {
            //     let offset = 62 + (level * 20);
            //     let bid_qty = i32::from_le_bytes(buffer[offset..offset+4]);
            //     ...
            // }
            Ok(MarketFeedMessage::Full { /* fields */ })
        }
        50 => {
            // Disconnect
            let reason = i16::from_le_bytes(buffer[8..10].try_into()?);
            Ok(MarketFeedMessage::Disconnect { reason })
        }
        _ => Err(ParseError::UnknownResponseCode(response_code)),
    }
}
```

> **IMPORTANT**: Byte offsets above are **0-based** (Rust). Dhan docs use 1-based. Conversion already applied.

## 13. Differences from Full Market Depth WebSocket

Do NOT confuse this WebSocket with the **Full Market Depth** WebSocket:

| Aspect               | Live Market Feed (this doc)          | Full Market Depth                      |
|-----------------------|--------------------------------------|----------------------------------------|
| Endpoint              | `wss://api-feed.dhan.co`            | 20-lvl: `wss://depth-api-feed.dhan.co/twentydepth`, 200-lvl: `wss://full-depth-api.dhan.co/twohundreddepth` |
| Depth levels          | 5 levels (in Full mode only)         | 20 or 200 levels                       |
| Header size           | 8 bytes                              | 12 bytes (different byte order!)       |
| Depth prices          | f32 (4 bytes)                        | f64 (8 bytes)                          |
| Use case              | General market data + basic depth    | Deep order book analysis               |
