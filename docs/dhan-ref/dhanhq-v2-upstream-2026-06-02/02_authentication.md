# Authentication — DhanHQ v2

> Source: https://dhanhq.co/docs/v2/authentication/ · API v2 · Audited 2 Jun 2026

Every request needs a JWT `access-token` header. Users are either **Individual** (build on your own Dhan account) or **Partner** (platforms serving many users; need `partner_id`/`partner_secret` from Dhan).

## Eligibility
- Trading APIs: free for all Dhan users.
- Data APIs: paid add-on (charges on platform).
- Partner onboarding: https://dhanhq.co/trading-apis.

## Access for individual traders
Two ways to get an access token: (1) generate directly from Dhan Web, or (2) API-key (OAuth) flow.

### Access token (manual, from Dhan Web)
1. Login to https://web.dhan.co
2. My Profile → **Access DhanHQ APIs**
3. Generate **Access Token** — **valid 24 hours**.
4. Optionally enter a Postback URL to receive order updates.

### Generate token via API (requires TOTP enabled)
```bash
curl --location --request POST \
'https://auth.dhan.co/app/generateAccessToken?dhanClientId=1000000001&pin=11111&totp=000000'
```
**Query params**
| Field | Description |
|---|---|
| `dhanClientId` | Client ID of the user |
| `pin` | Dhan PIN — 6-digit numeric |
| `totp` | TOTP from authenticator |

**Response**
```json
{
  "dhanClientId": "1000000401",
  "dhanClientName": "JOHN DOE",
  "dhanClientUcc": "ABCD12345E",
  "givenPowerOfAttorney": false,
  "accessToken": "eyJ...",
  "expiryTime": "2026-01-01T00:00:00.000"
}
```
| Field | Type | Description |
|---|---|---|
| `dhanClientId` | string | User ID generated by Dhan |
| `dhanClientName` | string | Registered name |
| `dhanClientUcc` | string | Unique Client Code |
| `givenPowerOfAttorney` | boolean | DDPI status |
| `accessToken` | string | Generated token |
| `expiryTime` | string | 24h from generation |

### Renew token (web-generated tokens only)
```bash
curl --location 'https://api.dhan.co/v2/RenewToken' \
  --header 'access-token: {JWT Token}' \
  --header 'dhanClientId: {Client ID}'
```
Expires the current token and issues a new 24h token. **Only renews active tokens** — renewing an expired token errors.

### API key & secret (OAuth, 3 steps)
Generate at Dhan Web → My Profile → Access DhanHQ APIs → **API key** toggle; enter **App name**, **Redirect URL**, optional **Postback URL**. **Key & secret valid 12 months.**

**Step 1 — Generate consent**
```bash
curl --location --request POST 'https://auth.dhan.co/app/generate-consent?client_id={dhanClientId}' \
  --header 'app_id: {API key}' \
  --header 'app_secret: {API secret}'
```
Returns `consentAppId` (needed for step 2). Up to **25 consentAppId/day**; each stays active until a `tokenId` is generated; only one token at a time.
**Headers:** `app_id` (required), `app_secret` (required).
**Response**
```json
{ "consentAppId": "940b0ca1-3ff4-4476-b46e-03a3ce7dc55d", "consentAppStatus": "GENERATED", "status": "success" }
```

**Step 2 — Browser login** (open directly in a browser; ends in 302 redirect)
```
https://auth.dhan.co/login/consentApp-login?consentAppId={consentAppId}
```
Redirects to `{redirect_URL}/?tokenId={Token ID}`. Consume `tokenId` from the path.

**Step 3 — Consume consent**
```bash
curl --location 'https://auth.dhan.co/app/consumeApp-consent?tokenId={Token ID}' \
  --header 'app_id: {API Key}' \
  --header 'app_secret: {API Secret}'
```
**Response**
```json
{
  "dhanClientId": "1000000001",
  "dhanClientName": "JOHN DOE",
  "dhanClientUcc": "CEFE4265",
  "givenPowerOfAttorney": true,
  "accessToken": "{access token}",
  "expiryTime": "2025-09-23T12:37:23"
}
```

## For partners (3 steps)
Once a partner has `partner_id` & `partner_secret`:

**Step 1 — Generate consent**
```bash
curl --location 'https://auth.dhan.co/partner/generate-consent' \
  --header 'partner_id: {Partner ID}' \
  --header 'partner_secret: {Partner Secret}'
```
Returns `consentId` (status `GENERATED`).

**Step 2 — Browser login for the user**
```
https://auth.dhan.co/consent-login?consentId={consentId}
```
302 redirect to `{redirect_URL}/?tokenId={Token ID}`.

**Step 3 — Consume consent**
```bash
curl --location 'https://auth.dhan.co/partner/consume-consent?tokenId={Token ID}' \
  --header 'partner_id: {Partner ID}' \
  --header 'partner_secret: {Partner Secret}'
```
Response is the same access-token payload shown above.

## Setup Static IP
Mandatory per SEBI/exchange guidelines, but **only for order-placement APIs** (Orders, Super Order, Forever Order). Fetching order/trade details needs no IP. Supports IPv4 & IPv6. Set a **PRIMARY** and a **SECONDARY** IP (each must be unique per user). **Once set, an IP can't be modified for 7 days.** Can also be managed at web.dhan.co.

### Set IP
```bash
curl --request POST --url https://api.dhan.co/v2/ip/setIP \
  --header 'Accept: application/json' --header 'Content-Type: application/json' \
  --header 'access-token: {Access Token}' \
  --data '{ "dhanClientId": "1000000001", "ip": "10.200.10.10", "ipFlag": "PRIMARY" }'
```
| Field | Type | Description |
|---|---|---|
| `dhanClientId` (req) | string | User ID |
| `ip` (req) | string | IPv4 or IPv6 |
| `ipFlag` (req) | enum | `PRIMARY` / `SECONDARY` |

Response: `{ "message": "IP saved successfully", "status": "SUCCESS" }`

### Modify IP
`PUT https://api.dhan.co/v2/ip/modifyIP` — same body; only allowed within the every-7-days window.

### Get IP
```bash
curl --request GET --url https://api.dhan.co/v2/ip/getIP \
  --header 'Accept: application/json' --header 'access-token: {Access Token}'
```
```json
{
  "modifyDateSecondary": "2025-09-30",
  "secondaryIP": "10.420.43.12",
  "modifyDatePrimary": "2025-09-28",
  "primaryIP": "10.420.29.14"
}
```
`modifyDate*` = date (YYYY-MM-DD) from which that IP can next be modified.

## Setup TOTP
**TOTP** = Time-based One-Time Password (RFC 6238): 6 digits from a shared secret + current time, refreshed every 30 s.
Setup: Dhan Web → DhanHQ Trading APIs → **Setup TOTP** → confirm with mobile/email OTP → scan QR / enter secret into an authenticator → confirm the first TOTP. Once enabled, TOTP appears as a login option on partner platforms and in the API-key flow, and unlocks token generation via API.

## User Profile
Validity / setup check — a good first integration test.
```bash
curl --location 'https://api.dhan.co/v2/profile' --header 'access-token: {JWT}'
```
```json
{
  "dhanClientId": "1100003626",
  "tokenValidity": "30/03/2025 15:37",
  "activeSegment": "Equity, Derivative, Currency, Commodity",
  "ddpi": "Active",
  "mtf": "Active",
  "dataPlan": "Active",
  "dataValidity": "2024-12-05 09:37:52.0"
}
```
| Field | Type | Description |
|---|---|---|
| `dhanClientId` | string | User ID |
| `tokenValidity` | string | Token validity date/time |
| `activeSegment` | string | Active segments |
| `ddpi` | string | `Active` / `Deactive` |
| `mtf` | string | `Active` / `Deactive` |
| `dataPlan` | string | Data-API subscription `Active` / `Deactive` |
| `dataValidity` | string | Data-API validity date/time |
