# Groww External-Floor Latency + NATS EOF — One-Page Summary (2026-07-03)

## What "external floor" means

Every Groww tick carries the server stamp `tsInMillis` (ms precision,
preserved into our tick store as the designated timestamp `ts`). We also
stamp `received_at` (our machine's receipt, same IST wall-clock basis).
`received_at − ts` is the end-to-end lag; the part we cannot engineer
away — Groww's server publish + network — is the **external floor**.

## The NATS EOF incident (Section B of the dossier)

**One-liner:** at 12:02:43 IST 2026-07-03, Groww's server closed our
NATS-over-WebSocket feed socket with no `-ERR` frame and no WS close
handshake (bare EOF, swallowed by nats-py into `last_error`; the SDK's
blocking `consume()` never returned); our stall watchdog force-closed
and the 50ms→5s reconnect ladder restored the feed.

**Status:** recovered automatically; verbatim CloudWatch lines pending
operator paste (AWS CLI unavailable in the build env); question set for
Groww engineering drafted. Not correlated with the 06:00 IST daily token
reset (event was ~6h later).
