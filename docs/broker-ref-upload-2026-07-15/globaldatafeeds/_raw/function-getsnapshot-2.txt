For example  : If user sends request for 1minute snapshot data at 9:20:01, he will get record of trade data between 9:19:00 to 9:20:00. However, if request is sent at 9:19:58 i.e. before minute is complete, user will get trade data between 9:18:00 to 9:19:00.                                                                                                                                                 In both cases, timestamp of the returned data will be start timestamp of the period. So when data between 9:15:00 to 9:20:00 is returned, it will have timestamp of 9:15:00.                                                                                                                                                                                                                                                                                                Also if a user subscribes for snapshot data of any symbol for a period and if no trades happen during that period, no data is sent by the server<\/p>\n\n<\/div>
