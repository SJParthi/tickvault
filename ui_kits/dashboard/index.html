06 · Composition Rules

A short recipe for building a new operator screen without drifting from the system.

Do

  • Start from #0d1117. Only step to #000 if the surface is explicitly "markets".
  • Right-align every number. First column is always left-aligned and brighter than the numerics.
  • Pair every status word with a colour (green/red/yellow) and, where live, a 2 s-pulse dot.
  • Use 14 px body, 11 px uppercase section headers with 2 px tracking. No headings above 28 px.
  • Use --tv-shadow-hover only on primary portal tiles, and only on hover.

Don't

  • Introduce a new gradient. The logo and avatar gradients are the only two in the system.
  • Use icon sets beyond the unicode glyphs + optional Lucide. Never mix sets.
  • Pad empty cells with placeholder prose — use -- dashes.
  • Animate list entries, fade routes, or add backdrop-filter blurs. Utilitarian motion only.
  • Use warm tones (amber-brown, peach, pastel). The system reads cool-black-and-blue with semantic green/red accents.
